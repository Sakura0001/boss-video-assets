# 目标学校名单

## 使用方式

本文件用于维护 Boss 自动招聘 Agent 的目标学校名单。Agent 在主动打招呼前，应读取候选人页面右侧显示的全部教育经历学校，并根据本文件判断是否继续触达。

## 匹配规则

- 以候选人页面右侧结构化教育经历中显示的学校为准。
- 本科通常显示一所学校；硕士或博士可能显示本科、硕士或博士等多所学校，只要其中任意一所命中即可通过学校门槛。
- 学校名称优先使用官方中文全称；海外院校可同时维护英文别名。
- 命中任一目标学校或别名，视为通过学校筛选。
- 自动匹配时，每条教育经历的学校字段必须与学校全称或批准别名完整相等；不得因为更长的
  院校名称包含目标学校文字而通过。例如“吉林大学交通学院”不得按“吉林大学”
  命中，只有独立显示“吉林大学”时才能通过。
- 高歧义简称不作为自动命中依据，例如“东大”“山大”“湖大”“中大”“西大”等。
- 未识别学校或学校不在名单内，默认不主动打招呼。
- 不要把候选人个人信息写入本文件。
- 不要在候选人沟通中暴露学校筛选逻辑。

## 自动化处理口径

### 学校在名单中

允许进入主动打招呼流程，并结合岗位知识库生成沟通话术。

### 学校不在名单中

默认跳过主动打招呼，不向候选人发送与学校筛选相关的解释。

### 学校无法识别

默认跳过主动打招呼；如后续需要更激进策略，可在 `config/school_policy.yaml` 中改为人工审核。

## 维护注意事项

- 本名单仅作内部招聘匹配策略使用。
- 不要在对候选人的自动回复中暴露学校筛选逻辑。
- 如名单来自内部政策，应保持版本更新，避免使用过期名单。
