# 候选人转化状态机

## 目标

把符合要求且具有转投资格的候选人推进到 Boss 微信交换动作完成。成功状态是 `wechat_exchange_completed`，不是候选人口头说会添加微信。

## 资格门槛

候选人必须同时满足：2027 届、本科或研究生、最终学历学校命中目标学校、理工科专业。技术经历和工作经历不是门槛。学校、专业、学历或毕业年份不符合或无法识别时，不回复。

## 主状态

`eligible → greeted → waiting_resume → resume_received → waiting_application_status → handoff_eligible → wechat_exchange_completed`

人工发送新消息时转为 `manual_takeover`。拒绝、无资格、达到跟进上限或命中流程终止条件时转为 `stopped`。

## 简历处理

候选人发起附件简历请求后：

1. 发送：`收到同学，我先看下你的简历，稍等一下。`
2. 执行 `boss action agree-resume`。
3. 重新运行 `boss chat "<姓名>" --strict`，确认 `简历获取状态: 已获取`。
4. 简短等待后发送：`我看了下，你的简历和我们这边方向挺匹配的。同学你这边是不是还没有投递呀？`
5. 状态设为 `waiting_application_status`，不提前交换微信。

附件未获取成功时，不声称已看简历，不进入投递状态判断。

## 投递状态追问

候选人表示投过或问“投过还能投吗”时发送：

`同学你投的哪个部门呀？简历编号有给过别人吗？`

如流程阶段仍不明确，继续自然询问心理测评是否失败、是否已经参加面试。简历编号是否给过别人只作沟通信息，最终资格由部门和流程阶段决定。

候选人回避时可再问一次：

`同学你还没说投没投递过呢，简历编号有给过别人嘛？`

投递状态总共询问两次。仍不回答时设为 `stopped/application_status_unanswered`，不交换微信。

## 转投决策优先级

1. 已投递 ICT 下云软件研发部：不能再次投递，礼貌结束。
2. 心理测评未通过：不能再次投递，礼貌结束。
3. 已参加过任一轮面试，无论通过、未通过或等待结果：不能转投，礼貌结束。
4. 未参加面试，且投递部门不是云软件研发部：可以转投并交换微信。
5. 其他部门笔试未通过，心理测评未失败且未参加面试：可以投递我们部门重新笔试，无等待期，并交换微信。
6. 只知道投过华为但说不清组织：先确认是否参加面试；未参加面试则交换微信，后续在微信确认部门。
7. 尚未投递：交换微信。

已投其他部门但面试状态不明时，先问流程阶段。不得仅凭“不是云软件研发部”跳过面试条件。

## 结构化决策

提取以下字段并调用 `runtime_store.py evaluate-transfer`：

- `application_target`: `none | cloud_software | other | unknown`
- `psych_status`: `not_taken | passed | failed | unknown`
- `interview_status`: `not_started | started | unknown`
- `written_status`: `not_taken | passed | failed | unknown`

只在结果为 `exchange_wechat` 时执行微信动作。

## 微信交换

1. 发送：`同学，沟通下来我感觉你和我们数据库方向还挺匹配的。如果你方便的话，可以加我微信，后续沟通会更及时一些，我也可以继续帮你跟一下流程。`
2. 执行 `boss action wechat`。
3. 发送：`辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名，我后续在微信上跟你继续沟通。`
4. 重新读取会话，确认交换动作完成。
5. 运行 `runtime_store.py wechat-complete`，写入长期台账和当日事件。

候选人或招聘方任一方发起并确认交换都算成功。交换完成后停止主动跟进；候选人仍在 Boss 询问时回复后续流程在微信跟踪。

## 标准招聘流程

`投递 → 心理测评 → 笔试 → 两轮技术面试 → 一轮主管面试 → 提交审批 → 发 offer`
