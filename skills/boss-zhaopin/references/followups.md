# 跟进和追问话术

## 候选人未回复

hello，想再跟你确认一下，你近期是否考虑新的工作机会？如果方便，我可以把岗位情况简单发你参考。

## 候选人表示感兴趣

好的同学，我这边主要是数据库业务相关团队，方向偏数据库内核和管理系统研发和测试。方便的话辛苦发我一份附件简历，我帮你进一步看下和数据库方向的匹配度。

## 候选人点击发送简历按钮

收到同学

推荐 CLI：

```bash
boss send --text "收到同学"
boss action agree-resume
boss chat <姓名> --strict
```

重新打开会话后确认 `简历获取状态: 已获取`。这一步不要补“我这边先帮你看下”等额外解释。

## 候选人问机会大不大

我看了下机会挺大的，只要你性格测评和笔试通过了，这个帮不了你，到后面面评不差的话offer概率还是很大的

推荐 CLI：

```bash
boss send --text "我看了下机会挺大的，只要你性格测评和笔试通过了，这个帮不了你，到后面面评不差的话offer概率还是很大的"
```

## 候选人重复问机会或质疑是不是骗他

不要复读上一条“机会挺大的”原句，避免像模板。

没骗你哈，我不会拿这个忽悠你。你这个背景确实可以往我们方向推，但测评、笔试和面试还是要自己过。我们先加个微信，后续投递和编号我在微信上跟你说。

推荐 CLI：

```bash
boss send --text "没骗你哈，我不会拿这个忽悠你。你这个背景确实可以往我们方向推，但测评、笔试和面试还是要自己过。我们先加个微信，后续投递和编号我在微信上跟你说。"
```

之后执行微信承接三步。

## 候选人问怎么投递

不要在 Boss 侧发送投递链接、官网入口或岗位入口选择。直接回复：

```text
可以的同学，我们先加个微信，后续投递和流程我在微信上跟你说，会更方便一点。
```

随后执行微信承接三步。

## 候选人表示还没投递

没事同学，那我们可以先加个微信，后续投递和流程我在微信上跟你说，会更方便一点。

随后执行微信承接三步。

## 候选人说简历编号没给过别人

可以的同学，那我们可以先加个微信，后续投递和简历编号这些我在微信上跟你说。

随后执行微信承接三步。

## 候选人说简历编号给过别人或有转投意向

如果不是ICT这边，也可以了解下我们这个方向。可以直接转投过来，我们先加个微信，后面我在微信上跟你说具体怎么处理。

随后执行微信承接三步。

## 候选人说笔试挂了或问能不能捞

可以的同学，可以投递我们部门重新笔试。我们先加个微信，后续我在微信上跟你说具体怎么处理。

随后执行微信承接三步。

## 直接微信承接三步

```bash
boss send --text "同学，沟通下来我感觉你和我们数据库方向还挺匹配的。如果你方便的话，可以加我微信，后续沟通会更及时一些，我也可以继续帮你跟一下流程。"
boss action wechat
boss send --text "辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名，我后续在微信上跟你继续沟通。"
```

## 投递状态确认后邀请微信沟通

使用前置条件：

- 候选人已经回复并表现出意向。
- 候选人表示未投递、简历编号没给过别人、有转投/改投/路由意向，或笔试挂了想重新走流程。
- 候选人已经投递到合适 ICT 部门，但明确希望我帮忙跟流程。
- 不要在刚打招呼、只补充岗位介绍、还没确认投递状态时执行 `boss action wechat`。

## 发送微信入口后的补充说明

辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名，我后续在微信上跟你继续沟通。

## 候选人要求更多岗位信息

可以的，这个岗位核心方向是数据库内核和管理系统研发和测试。你比较关注地点、作息、面试流程还是方向优势？我可以重点给你说明。

## 候选人暂不考虑

好的，理解。后续如果你考虑新的机会，也欢迎随时联系我。祝你顺利。
