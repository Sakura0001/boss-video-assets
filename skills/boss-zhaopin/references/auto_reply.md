# 自动回复

1. 使用 `boss list --unread` 获取未读会话。
2. 逐个执行 `boss chat "<姓名>" --strict`，读取完整可见对话。
3. 回复前校验 2027 届、最终学历目标学校、能明确对应 `school_policy.yaml` 的 `allowed_majors` 方向的专业、本科或研究生；任一不符合或无法识别都不回复。
4. 检测用户手动发送的新消息；存在时标记 `manual_takeover`，等待候选人再次回复。
5. 候选人请求附件简历时按 `candidate_conversion.md` 接收并验证。
6. 候选人问机会时使用 `followups.md` 的固定首问或重复问分支。
7. 候选人询问投递链接或招聘/秋招是否开始时，使用 `application.md` 中对应岗位的官网、步骤和固定回复；方向不明时先问软件开发/测试、AI 或网络安全方向，随后发送微信交换请求。
8. 候选人谈投递、转投、测评、笔试或面试时，提取结构化字段并调用 `evaluate-transfer`。
9. 普通问题只使用已安装 references 中明确的答案。
10. 知识库没有答案时，不发送；记录 `knowledge_gap` 的脱敏摘要并提示用户。
11. 每次成功动作后记录对应事件，失败动作不计数。

交换微信、附件接收、投递终止等状态动作后必须重新读取会话验证。
