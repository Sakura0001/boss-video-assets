# 按需自动招聘运行

用户显式要求启动自动招聘后，按以下顺序执行，不逐条请求确认：

1. 初始化并清理本地状态。
2. 用 `boss list --unread` 验证登录；需要时执行 `boss login`，登录完成后继续。
3. 扫描全部未读会话，先校验候选人资格，再回复。
4. 查询并处理已经满六小时的跟进任务。
5. 查询当日招呼数，在剩余 150 人额度内处理推荐候选人。
6. 对每个会话推进到等待、人工接管、终止或 `wechat_exchange_completed`。
7. 更新长期去重、微信交换台账和事件。
8. 输出本次摘要；用户需要某日数据时运行 `daily-report --date YYYY-MM-DD`。

暂停整次运行的条件：登录失效且未恢复、验证码、风控、异常弹窗、CLI 连续失败。知识库缺失只暂停相关候选人并提示用户处理。
