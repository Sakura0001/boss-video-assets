# 投递与转投口径

Boss 侧只发送本文件维护的华为校招官网和投递步骤，不补充或猜测其他入口。候选人方向明确时只发送对应岗位的步骤；方向不明确时先询问软件开发/测试、AI 或网络安全方向，不要把三套步骤一次全部发送。

## 询问投递链接或招聘时间

候选人询问现在招聘、秋招是否已经开始，但没有说明岗位方向时，回复：

`同学已经开始了哈，现在可以投递。你想投软件开发/测试、AI还是网络安全方向呀？我把对应的投递步骤发你。`

候选人已经说明方向，或直接询问某类岗位的投递链接时，跳过上述追问，发送下面对应的一条固定回复。

### 软件测试类岗位/通用软件开发岗位

`同学，软件测试类岗位/通用软件开发岗位从校招官网投递：https://career.huawei.com。进入“校园招聘 → 应届生职位 → 软件开发工程师 → 通用软件/数据库”，部门意向选择“ICT BG”“云软件研发部”。投递后，把简历编码发给我即可。可以先加我微信，我后续帮你跟下流程。`

### AI 类岗位

`同学，AI类岗位从校招官网投递：https://career.huawei.com/reccampportal/portal5/campus-recruitment.html。进入“校园招聘 → 应届生职位 → AI应用工程师 → AI技术应用/AI系统软件”，部门意向选择“ICT BG”“云软件研发部”。投递后，把简历编码发给我即可。可以先加我微信，我后续帮你跟下流程。`

### 网络安全岗位

`同学，网络安全岗位从校招官网投递：https://career.huawei.com。进入“校园招聘 → 应届生职位 → 网络安全与隐私保护工程师 → 网络安全”，部门意向选择“ICT BG”“云软件研发部”。投递后，把简历编码发给我即可。可以先加我微信，我后续帮你跟下流程。`

发送招聘开始回复或对应岗位投递步骤后，立即执行 `boss action wechat` 发送微信交换请求。上述回复替代通用微信交换流程的前置话术，不再重复发送加微信邀请；动作后按 `candidate_conversion.md` 的剩余步骤重新读取会话，验证并记录交换结果。

## 未投递

回复：`可以的同学，那我们先加个微信，后续投递和流程我在微信上跟你说。`

随后执行微信交换流程。

## 已投非云软件研发部且未面试

回复：`可以的同学，只要还没有参加过面试，就可以转投我们这边。我们先加个微信，后续我在微信上跟你说具体怎么处理。`

随后执行微信交换流程。

## 其他部门笔试未通过

心理测评未失败且没有参加过面试时回复：

`可以的同学，可以投递我们部门重新笔试，没有等待期。我们先加个微信，后续我在微信上跟你说具体怎么处理。`

## 已投云软件研发部

回复：`同学，如果已经投递过ICT下的云软件研发部，就没办法再投递我们部门了。祝你后续流程顺利哈。`

结束联系，不交换微信。

## 心理测评失败或已经面试

回复：`同学，这种情况就没办法再转投我们部门了。祝你后续求职顺利哈。`

结束联系，不交换微信。

## 组织说不清

先确认是否参加过面试。没有参加面试时回复：

`可以的同学，具体部门我们到微信上再确认一下，你先加我微信，后续我跟你说怎么处理。`

## 交换完成后

候选人仍在 Boss 问投递流程时，按岗位方向发送本文件对应的投递步骤；已经发送过时回复：`同学，后续投递和流程我们在微信上跟踪哈，投递后把简历编码发给我即可。`
