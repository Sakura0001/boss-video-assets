# 工作流：每日自动化

## 目标

将主动打招呼和自动回复组合成每日可执行流程。

## 建议顺序

1. 检查 Boss 登录态。
2. 读取职位列表：`boss positions`。
3. 对默认岗位执行主动打招呼流程。
4. 对已触达且有回复的候选人，按 `workflows/candidate_conversion.md` 推进：判断意向、索要简历；候选人已发简历或沟通差不多时，发送微信入口并提醒候选人主动添加微信。
5. 每隔 `poll_unread_interval_seconds` 秒执行一次自动回复流程。
6. 达到每日自动打招呼或自动回复上限后停止对应动作。
7. 输出执行摘要。

## 暂停条件

- Boss 登录失效。
- 平台出现风控、验证码、异常弹窗。
- 知识库缺失严重，无法生成可靠回复。
- 自动发送被合规检查连续拦截。
- 用户手动关闭全自动开关。

## 人工维护点

- 每日触达上限：`config/agent.yaml`
- 默认岗位：`config/agent.yaml`
- 风险策略：`config/risk_policy.yaml`
