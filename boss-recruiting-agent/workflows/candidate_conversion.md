# 工作流：候选人转化标准流程

## 目标

把 Boss 直聘上的候选人从“推荐列表”推进到“微信沟通”。流程结束点是候选人已发送微信或已明确知道需要主动添加我的微信，后续沟通转到微信完成。

## 根本要求

- 全程使用招聘负责人本人的口吻，不要暴露 AI、Agent、机器人、自动化或知识库。
- 消息要像真实 HR 在即时沟通，不要像批量群发。
- 只在候选人学校命中目标名单后主动触达。
- 候选人没有表现出意向前，不要索要微信。
- 发出微信交换入口后，必须补充一句：请候选人主动添加我，不要由我频繁主动添加候选人，避免微信异常。

## 标准状态机

### 1. 学校筛选

1. 使用 `boss recommend <岗位关键字>` 获取候选人。
2. 识别候选人最终学历学校。
3. 对照 `knowledge/schools/target-schools.md`。
4. 学校命中时进入下一步；学校未命中或无法识别时默认跳过。

### 2. 主动打招呼

使用 `boss greet <姓名>` 对候选人打招呼。

打招呼内容参考：

```text
hello，我这边是华为数据库业务相关团队，主要做 TaurusDB、RDS、GaussDB、机密数据库、Redis 等方向。看你背景和数据库/软件研发方向比较匹配，想和你简单沟通一下，方便了解机会吗？
```

### 3. 依次补充岗位信息

打招呼后，如果会话允许继续发送消息，按顺序补充 3-4 条信息。每条单独发送，不要合成一大段。

建议顺序：

1. 部门和产品方向。
2. 岗位工作内容。
3. 岗位优势。
4. 城市和后续沟通方式。

CLI 示例：

```bash
boss send --text "我这边主要是华为数据库业务相关团队，方向包括 TaurusDB、RDS、GaussDB、机密数据库、Redis 等。"
boss send --text "岗位主要偏数据库/大数据/数据湖仓内核和管理系统研发，会涉及存储引擎、查询引擎、事务引擎、备份恢复、权限管理、监控诊断等方向。"
boss send --text "数据库本身是比较垂直的方向，技术深度会比普通业务开发更深一点。这个方向短期内不太容易被大模型直接替代，后面也更容易建立自己的技术护城河。"
boss send --text "目前办公地点主要有上海、杭州、西安、东莞、北京、南京，具体城市可以结合岗位 HC 和你的意向再确认。"
```

### 4. 候选人回复后判断意向

如果候选人回复并表现出意向，例如：

- “可以了解”
- “感兴趣”
- “方便”
- “岗位还在吗”
- “怎么投”
- “可以发 JD 吗”
- “薪资/地点/流程是什么”

则进入索要简历流程。

如果候选人明确拒绝或暂不考虑，则礼貌结束，不继续追问微信。

### 5. 索要简历

候选人有意向后，先索要附件简历。

优先使用：

```bash
boss send --text "方便的话辛苦发我一份附件简历，我帮你进一步看下和数据库方向的匹配度。" --request-resume
```

如果当前会话已经适合直接触发工具栏“求简历”，也可以使用：

```bash
boss action request-attachment-resume
```

注意：

- 不要在候选人没有意向前索要简历。
- 不要要求候选人提供与招聘无关的隐私信息。
- 简历原文和候选人隐私不要提交到 Git。

### 6. 候选人发送简历后询问是否愿意了解部门

候选人发送简历后，发送：

```bash
boss send --text "同学，收到，我先帮你看一下。你这边是否有意向来我们数据库部门了解一下？如果方便的话，可以加我微信，后续沟通会更及时一些。"
```

### 7. 发送微信交换入口

调用 Boss CLI 的微信动作：

```bash
boss action wechat
```

发送微信动作后，必须补充：

```bash
boss send --text "辛苦你这边主动加我一下哈，我这边如果频繁主动添加同学，微信容易异常。加上后你备注一下姓名和数据库方向，我后续在微信上跟你继续沟通。"
```

### 8. 结束 Boss 对话

当候选人发来微信、表示已添加，或已明确知道需要主动添加微信后，Boss 会话可以结束。后续沟通转到微信，不再在 Boss 中继续重复推进。

## 结束条件

- 候选人已发送微信。
- 候选人已表示会主动添加微信。
- 候选人明确拒绝或暂不考虑。
- 候选人学校未命中目标名单。
- 候选人没有回复，等待后续定时跟进。
