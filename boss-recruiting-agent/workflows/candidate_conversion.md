# 工作流：候选人转化标准流程

## 目标

把 Boss 直聘上的候选人从“推荐列表”推进到“微信沟通”。流程结束点是候选人已发送微信或已明确知道需要主动添加我的微信，后续沟通转到微信完成。

## 根本要求

- 全程使用招聘负责人本人的口吻，不要暴露 AI、Agent、机器人、自动化或知识库。
- 消息要像真实 HR 在即时沟通，不要像模板话术。
- 只在候选人学校命中目标名单后主动触达。
- 候选人没有表现出意向前，不要索要微信。
- `boss greet` 后，平台会自动发送默认招呼语；Agent 只负责立刻补充真人化介绍和岗位优势，不要直接推进到微信。
- 候选人回复后，必须先确认投递状态和投递部门，再决定是否进入微信承接。
- 候选人未回复、未确认投递状态、或未确认需要我们承接到“云软件研发部”前，不要执行 `boss action wechat`。
- 候选人已经表达兴趣、投递路径需要我们承接且沟通差不多时，要及时调用 Boss CLI 的微信交换动作，不要在 Boss 中继续拖长对话。
- 发出微信交换入口后，必须补充一句：请候选人主动添加我，不要由我频繁主动添加候选人，因为账号一天不能主动添加太多同学，频繁添加容易异常。

## 标准状态机

### 1. 学校筛选

1. 使用 `boss recommend <岗位关键字>` 获取候选人。
2. 识别候选人最终学历学校。
3. 对照 `knowledge/schools/target-schools.md`。
4. 学校命中时进入下一步；学校未命中或无法识别时默认跳过。

### 2. 主动打招呼

使用 `boss greet <姓名>` 对候选人打招呼。

Boss 点击打招呼后会自动发送一条平台招呼语。这个动作之后，必须立刻用 `boss send` 补充一条真人化消息，说明我看过候选人背景和岗位方向匹配，避免平台默认招呼语显得过于模板化。

必须补充的话术：

```text
我看了下你的背景，和我们数据库/软件研发/测试开发方向还挺匹配的，方便的话我给你简单介绍下我们这边方向。
```

CLI：

```bash
boss send --text "我看了下你的背景，和我们数据库/软件研发/测试开发方向还挺匹配的，方便的话我给你简单介绍下我们这边方向。"
```

### 3. 依次补充岗位信息

真人化补充消息发送后，如果会话允许继续发送消息，只补充 2 条岗位介绍。每条单独发送，不要合成一大段，也不要继续连发 4-5 条。城市、流程、投递等信息等候选人回复或主动询问后再说。

建议顺序：

1. 部门/产品方向 + 岗位工作内容。
2. 开发/AI 开发方向 + 岗位优势。

CLI 示例：

```bash
boss send --text "我看了下你的背景，和我们数据库/软件研发/测试开发方向还挺匹配的，方便的话我给你简单介绍下我们这边方向。"
boss send --text "我这边主要是华为数据库业务相关团队，方向包括TaurusDB、RDS、GaussDB、GeminiDB、Redis等，岗位主要偏数据库/大数据/数据湖仓内核和管理系统研发。"
boss send --text "数据库方向技术深度会比普通业务开发更深一点，后面也更容易建立自己的技术护城河。同时我们这边因为公司长期投入，很适合员工来长期培养，数据库没有受到当下AI的太多冲击，所以希望同学考虑下我们这里。"
```

### 4. 等候选人回复

打招呼和两条岗位介绍发送完后，主动索要附件简历：

```bash
boss send --text "方便的话辛苦发我一份附件简历，我帮你进一步看下和数据库方向的匹配度。" --request-resume
```

索要简历后先等待候选人回复。候选人没有回复前，不要索要微信，不要执行 `boss action wechat`。

候选人明确回复并表现出意向后，再进入下一步。

如果候选人先发起附件简历请求，消息中出现“对方想发送附件简历给您，您是否同意”，必须先接收简历：

```bash
boss action agree-resume
boss chat <姓名> --strict
```

重新打开会话后确认 `简历获取状态: 已获取`，再进入后续投递状态确认、投递引导或微信承接。不要在未接收简历时直接推进后续环节。

### 5. 候选人回复后先判断投递状态

候选人回复后，正式引导投递或微信承接前，先自然确认：

```bash
boss send --text "同学你这边是不是还没有投递呀？"
```

处理规则：

- 候选人还没投递：可以继续介绍我们这边方向，并引导官网投递，投递页面选择“云软件研发部”。
- 候选人已经投递到云软件研发部或合适的 ICT 部门：不要继续重复推投递，避免骚扰；如候选人仍希望我跟流程，可以再进入微信承接。
- 候选人已经投递但不是云软件研发部或合适的 ICT 部门：可以继续介绍云软件研发部方向，说明如果感兴趣可以看看后续有没有路由机会；此时可以进入微信承接。
- 候选人笔试没过：可以说明“我们这边可以再帮你看看有没有二次笔试机会”，但不要承诺一定能安排。

### 6. 候选人回复后判断意向

如果候选人回复并表现出意向，例如：

- “可以了解”
- “感兴趣”
- “方便”
- “岗位还在吗”
- “怎么投”
- “可以发 JD 吗”
- “薪资/地点/流程是什么”

则进入投递引导流程；如果候选人已发起附件简历请求，先按第 4 步接收简历。只有已经完成第 5 步投递状态确认，并且候选人未投递、投递的不是云软件研发部、需要路由到我们部门、或明确希望我帮忙跟流程时，才可以进入微信承接流程。

如果候选人明确拒绝或暂不考虑，则礼貌结束，不继续追问微信。

### 7. 索要简历

本流程默认在两条岗位介绍之后已经主动索要附件简历。候选人后续表现出意向但尚未发送简历时，再次轻量提醒一次即可，不要反复催促。

优先使用：

```bash
boss send --text "方便的话辛苦发我一份附件简历，我帮你进一步看下和数据库方向的匹配度。" --request-resume
```

如果当前会话已经适合直接触发工具栏“求简历”，也可以使用：

```bash
boss action request-attachment-resume
```

注意：

- 主动触达流程中，岗位介绍完成后可以主动索要一次附件简历。
- 如果候选人已经先发起附件简历请求，必须先执行 `boss action agree-resume` 接收简历，并确认 `简历获取状态: 已获取`。
- 不要要求候选人提供与招聘无关的隐私信息。
- 简历原文和候选人隐私不要提交到 Git。

### 8. 引导官网投递和记录编号

候选人确认有意向投递时，优先按 `knowledge/application/apply-links.md` 引导投递。投递页面必须明确提醒候选人选择“云软件研发部”，这样简历可以路由到我们部门，后续才方便跟踪。不要说“按实际情况选”“这个都可以”等模糊口径。

投递完成后，必须让候选人把简历编号/简历编码发回来。收到每一个简历编号后，必须做明确记录，至少记录候选人、简历编号、投递岗位、部门选择、收到时间和当前跟进状态。记录放在本地运行记录或外部招聘台账中，不提交到 Git，不保存简历原文、手机号、微信号等无关隐私。

如果候选人主动表示已经在官网投递，统一回复：

```bash
boss send --text "可以的同学，你方便的话把官网投递后的简历编号/简历编码也发我一下，我这边帮你备案看看，不一定能成功，最终还是以系统流程和面试匹配为准哈。"
```

候选人问投递选什么时：

```bash
boss send --text "同学，投递的时候选择云软件研发部就可以。这样简历可以直接路由到我们部门，我这边后面也能帮你跟。"
boss send --text "投完之后把简历编号发我一下哈，我这边帮你做个备案。简历编号尽量别到处发给多个部门或者多个HR，不然流程可能会被其他地方占住，我们部门这边就不好推进了，也容易耽误你秋招。"
```

### 9. 投递状态确认后转微信

只有满足以下条件之一，才可以发送微信交换话术并执行 `boss action wechat`：

- 候选人回复后已确认还没有投递，并且愿意了解或准备投递云软件研发部。
- 候选人已投递，但投递的不是云软件研发部或合适的 ICT 部门，需要我帮忙确认路由机会。
- 候选人已投递云软件研发部或合适的 ICT 部门，并明确希望我帮忙跟流程。
- 候选人笔试没过，正在了解我们这边是否有二次笔试或重新承接机会。

禁止在以下情况下换微信：

- 刚执行 `boss greet`，候选人还没有回复。
- 只补充了岗位介绍和优势，但还没有确认投递状态。
- 候选人已经投递到合适部门且没有表达需要我跟流程。

交换微信后，后续沟通转到微信承接，Boss 侧不再继续反复推进投递细节。

```bash
boss send --text "同学，沟通下来我感觉你和我们数据库方向还挺匹配的。如果你方便的话，可以加我微信，后续沟通会更及时一些，我也可以继续帮你跟一下流程。"
```

### 10. 发送微信交换入口

调用 Boss CLI 的微信动作：

```bash
boss action wechat
```

发送微信动作后，必须补充：

```bash
boss send --text "辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名和数据库方向，我后续在微信上跟你继续沟通。"
```

### 11. 结束 Boss 对话

当候选人发来微信、表示已添加，或已明确知道需要主动添加微信后，Boss 会话可以结束。后续沟通转到微信，不再在 Boss 中继续重复推进。

## 结束条件

- 候选人已发送微信。
- 候选人已表示会主动添加微信。
- 候选人明确拒绝或暂不考虑。
- 候选人学校未命中目标名单。
- 候选人没有回复，等待后续定时跟进。
