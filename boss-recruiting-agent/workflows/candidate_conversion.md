# 工作流：候选人转化标准流程

## 目标

把 Boss 直聘上的候选人从“推荐列表/会话”推进到“微信沟通”。流程结束点是候选人已发送微信、已添加微信，或已明确知道需要主动添加我的微信；后续投递、编号、转投、笔试重走等细节都在微信完成。

## 根本要求

- 全程使用招聘负责人本人的口吻，不要暴露 AI、Agent、机器人、自动化或知识库。
- 消息要像真实 HR 在即时沟通，不要像模板话术。
- 只在候选人学校命中目标名单后主动触达。
- 候选人没有回复或没有表现出意向前，不要索要微信。
- Boss 侧不再维护或发送投递链接、官网入口、岗位入口选择、真实 URL 或简历编号备案细节。
- 候选人表示未投递、简历编号没给过别人、有转投/改投/路由意向，或笔试挂了想重新走流程时，直接进入微信承接。
- 发出微信交换入口后，必须补充一句：请候选人主动添加我，不要由我频繁主动添加候选人，因为账号一天不能主动添加太多同学，频繁添加容易异常。

## 标准状态机

### 1. 学校筛选

1. 使用 `boss recommend <岗位关键字>` 获取候选人。
2. 识别候选人最终学历学校。
3. 对照 `knowledge/schools/target-schools.md`。
4. 学校命中时进入下一步；学校未命中或无法识别时默认跳过。

### 2. 主动打招呼

使用 `boss greet <姓名>` 对候选人打招呼。

Boss 点击打招呼后会自动发送一条平台招呼语。这个动作之后，必须立刻用 `boss send` 补充一条真人化消息，说明我看过候选人背景和岗位方向匹配。

```bash
boss send --text "我看了下你的背景，和我们数据库方向挺匹配的，方便的话我给你简单介绍下我们这边方向。"
```

### 3. 依次补充岗位信息

真人化补充消息发送后，如果会话允许继续发送消息，只补充 1 条合并岗位介绍。岗位介绍一次发完，不要拆成两条占用发送机会，也不要继续连发 4-5 条。城市、流程、投递等信息等候选人回复或主动询问后再说。

```bash
boss send --text "我这边主要是华为数据库业务相关团队，方向包括TaurusDB、RDS、GaussDB、GeminiDB、Redis等，岗位主要偏数据库内核和管理系统研发和测试。数据库方向技术深度会比普通业务开发更深一点，后面也更容易建立自己的技术护城河。同时我们这边因为公司长期投入，很适合员工来长期培养，数据库没有受到当下AI的太多冲击，所以希望同学考虑下我们这里。"
```

### 4. 索要附件简历并等待回复

```bash
boss send --text "方便的话辛苦发我一份附件简历，我帮你进一步看下和数据库方向的匹配度。" --request-resume
```

索要简历后先等待候选人回复。候选人没有回复前，不要索要微信，不要执行 `boss action wechat`。

如果候选人先发起附件简历请求，消息中出现“对方想发送附件简历给您，您是否同意”，先只回复“收到同学”，然后接收简历：

```bash
boss send --text "收到同学"
boss action agree-resume
boss chat <姓名> --strict
```

重新打开会话后确认 `简历获取状态: 已获取`，再进入后续投递状态确认或微信承接。

### 5. 候选人回复后判断是否直接微信承接

如果候选人没有明确投递状态，可以先问：

```bash
boss send --text "同学你这边是不是还没有投递呀？"
```

以下任一情况出现时，直接进入微信承接，不要在 Boss 侧继续发送投递链接或投递细节：

- 候选人表示还没投递过。
- 候选人表示简历编号还没给过别人。
- 候选人问怎么投递、投哪个岗位、投递入口是什么。
- 候选人表示想转投、改投、路由到我们方向，或简历编号给过其他部门但愿意了解我们方向。
- 候选人笔试没过，询问能不能重新投递或重新笔试。
- 候选人已投递到合适 ICT 部门，但明确希望我帮忙跟流程。

如果候选人已经投递到合适 ICT 部门，且没有表达需要我帮忙跟流程，不要重复推投递，也不要强行换微信。

### 6. 直接微信承接三步

```bash
boss send --text "看你简历我感觉你和我们数据库方向还挺匹配的"
boss action wechat
boss send --text "辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名，我后续在微信上跟你继续沟通。"
```

微信入口发出后，Boss 侧不再继续反复推进投递、编号、链接或入口选择细节。

### 7. 常见分支话术

候选人还没投递：

```bash
boss send --text "没事同学，那我们可以先加个微信，后续投递和流程我在微信上跟你说，会更方便一点。"
```

候选人简历编号没给过别人：

```bash
boss send --text "可以的同学，那我们可以先加个微信，后续投递和简历编号这些我在微信上跟你说。"
```

候选人简历编号给过别人或有转投意向：

```bash
boss send --text "如果不是ICT这边，也可以了解下我们这个方向。可以直接转投过来，我们先加个微信，后面我在微信上跟你说具体怎么处理。"
```

候选人笔试挂了或问能不能捞：

```bash
boss send --text "可以的同学，可以投递我们部门重新笔试。我们先加个微信，后续我在微信上跟你说具体怎么处理。"
```

候选人问“机会大吗”“进的机会大吗”“概率大吗”等：

```bash
boss send --text "我看了下机会挺大的，只要你性格测评和笔试通过了，这个帮不了你，到后面面评不差的话offer概率还是很大的"
```

候选人重复问机会，或追问“你没骗我吧”“真的假的”等：

```bash
boss send --text "没骗你哈，我不会拿这个忽悠你。你这个背景确实可以往我们方向推，但测评、笔试和面试还是要自己过。我们先加个微信，后续投递和编号我在微信上跟你说。"
```

### 8. 结束条件

- 候选人已发送微信。
- 候选人已表示会主动添加微信。
- 候选人明确拒绝或暂不考虑。
- 候选人学校未命中目标名单。
- 候选人没有回复，等待后续定时跟进。
