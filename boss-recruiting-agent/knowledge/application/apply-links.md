# 投递状态和微信承接口径

本文件不再维护投递链接、真实 URL 或岗位链接表。Boss 侧不要给候选人发送投递链接、官网入口、岗位入口选择细节或简历编号备案说明；这些内容统一转到微信后由招聘负责人继续沟通。

## 判断原则

正式推进前，先自然确认候选人投递状态：

```text
同学你这边是不是还没有投递呀？
```

以下任一情况出现时，直接进入微信承接：

- 候选人表示还没投递过。
- 候选人表示简历编号还没给过别人。
- 候选人表示想转投、改投、路由到我们方向，或简历编号给过其他部门但愿意了解我们方向。
- 候选人笔试没过，询问是否能重新投递或重新笔试。

如果候选人已经投递到合适的 ICT 部门，且没有表达需要我帮忙跟流程，不要重复推投递，也不要强行换微信。

## 直接微信承接

先发送：

```text
看你简历我感觉你和我们数据库方向还挺匹配的
```

然后执行：

```bash
boss action wechat
```

最后补充：

```text
辛苦你这边主动加我一下哈，我们账号一天不能主动添加太多同学，频繁添加容易异常。加上后你备注一下姓名，我后续在微信上跟你继续沟通。
```

## 常见分支

### 候选人还没投递

```text
没事同学，那我们可以先加个微信，后续投递和流程我在微信上跟你说，会更方便一点。
```

随后执行微信承接三步。

### 候选人简历编号没给过别人

```text
可以的同学，那我们可以先加个微信，后续投递和简历编号这些我在微信上跟你说。
```

随后执行微信承接三步。

### 候选人简历编号给过别人或有转投意向

```text
如果不是ICT这边，也可以了解下我们这个方向。可以直接转投过来，我们先加个微信，后面我在微信上跟你说具体怎么处理。
```

随后执行微信承接三步。

### 候选人笔试没过

```text
可以的同学，可以投递我们部门重新笔试。我们先加个微信，后续我在微信上跟你说具体怎么处理。
```

随后执行微信承接三步。

### 候选人已经投递到合适 ICT 部门

```text
如果你已经投递到合适的ICT部门了，那就正常等流程就行，我这边就不重复打扰你投递。
```

如果候选人明确希望我帮忙跟流程，再进入微信承接。

## 维护规则

- 不要在本文件维护投递链接、真实 URL 或岗位链接表。
- 不要在 Boss 侧发送投递链接、官网入口或具体岗位入口。
- 不要在 Boss 侧继续反复推进投递细节；满足承接条件后直接交换微信。
- 不要把候选人的简历编号、手机号、微信号或其他隐私信息写入本文件。
